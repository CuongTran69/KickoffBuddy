## MVP: Không cần BE, không cần server DB

Tất cả data trong MVP đều là **local-only**:

| Loại data | Lưu ở đâu | Package |
|---|---|---|
| Match list, reminders | Local DB trên thiết bị | Isar hoặc Hive |
| User preferences, timezone, football level | SharedPreferences | `shared_preferences` |
| Rule cards, vocabulary | JSON bundle đóng gói trong app | assets/ |
| No Spoiler state | Local DB | Isar/Hive |

**Notification** cũng local — `flutter_local_notifications` schedule trực tiếp trên thiết bị, không cần server push.

---

## Ngoại lệ duy nhất: Magic Add gọi LLM API

Đây là điểm **duy nhất** cần network trong MVP, và chỉ khi parser offline không đủ confidence. Không cần BE riêng — gọi thẳng từ app đến LLM API (Claude Haiku hoặc GPT-4o-mini).

Tuy nhiên có một vấn đề cần quyết định:

**API key để ở đâu?**

| Option | Ưu | Nhược |
|---|---|---|
| A. Hardcode trong app | Đơn giản, không cần BE | Key bị lộ nếu decompile app — **không nên** |
| B. ★ Proxy BE nhỏ (1 endpoint) | Key an toàn, có thể rate limit | Cần deploy 1 server nhỏ |
| C. User tự nhập API key | Không cần BE, key an toàn | UX xấu, không phù hợp người dùng phổ thông |

Recommendation: **Option B** — một proxy endpoint đơn giản (Cloudflare Worker hoặc Supabase Edge Function) nhận text từ app, gọi LLM, trả về JSON. Không cần DB, không cần auth phức tạp, deploy miễn phí.
