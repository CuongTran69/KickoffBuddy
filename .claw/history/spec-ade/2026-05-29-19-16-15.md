### 🟢 Nhóm 1 — Có thể build ngay, offline, không cần backend

Đây là những feature **đã có data sẵn** hoặc **pure UI/logic**, build được trong 1-2 ngày:

#### A. Sleep Plan (★ Gợi ý cao nhất)
Đơn giản nhất trong tất cả. Logic thuần túy: lấy `kickoffAtUtc` → convert sang local time → nếu giờ khuya (22:00–05:00) → hiển thị 3 gợi ý (Late Watcher / Balanced / Healthy Replay). Không cần API, không cần backend. Tích hợp tự nhiên vào Match Detail screen hoặc Home hero card. Data đã có sẵn trong Isar.

```
Trận 02:00 sáng → hiện card:
┌─────────────────────────────┐
│ 🌙 Trận khuya — Kế hoạch?  │
│ ● Thức xem live (Late)      │
│ ● Xem hiệp 2 (Balanced)     │
│ ● Ngủ + Replay sáng (Healthy)│
└─────────────────────────────┘
```

#### B. Fan Etiquette Guide
Content đã viết xong trong `docs/future/03-fan-etiquette.md` — 7 chủ đề đầy đủ. Chỉ cần build UI tương tự Rule Cards (list + detail). Có thể dùng lại `RuleCardRepository` pattern, chỉ đổi data source. Effort: ~1 ngày.

#### C. Replay Planner: nút "Đã xem"
Đã deferred từ MVP nhưng logic đơn giản: thêm button "Đã xem" vào Match Detail khi `replayPlannerEnabled == true` → set `replayPlannerEnabled = false`, cancel notification, hiển thị recap. Tất cả infrastructure đã có.

#### D. Cảnh báo đêm khuya trong Match Scheduler
Khi user thêm trận thủ công với giờ 22:00–05:00 → hiện warning "Trận này diễn ra lúc 2 giờ sáng. Bạn có muốn bật Replay Planner không?". Pure UI, không cần gì thêm.

### 🟡 Nhóm 2 — Cần thêm data/content, không cần backend

#### E. "Why Did That Happen" (Rule Cards)
10 tình huống tra cứu (VAR hủy bàn, penalty, thẻ đỏ...). Cần viết content JSON, sau đó build UI. Effort: content ~2 ngày, UI ~1 ngày. Có thể tái dùng Rule Card detail screen.

#### F. After Match Learning (Quiz lite)
Sau khi trận kết thúc (status = `completed`), hiện prompt "Bạn gặp tình huống nào khó hiểu?" → link đến Rule Card tương ứng. Không cần quiz engine thực sự — chỉ là deep-link vào Rule Cards. Effort: ~1 ngày.

#### G. Badge System (4 badge MVP)
Tính từ data đã có trong Isar: số rule cards đã xem, số vocabulary terms, số trận Replay Planner. Không cần backend. Effort: ~1 ngày logic + UI.
